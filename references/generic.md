# Generic — Универсальные паттерны

## Минимальные требования

Любая агентная система должна иметь:

1. **Память** — персистентное хранение между сессиями
2. **Навыки** — модульные расширения возможностей
3. **Команды** — способ взаимодействия

## Структура памяти

```
~/.agent/
├── MEMORY.md           # Долгосрочная память
├── memory/             # Дневники
│   └── YYYY-MM-DD.md
└── config.json         # Конфигурация
```

## Навыки

### Минимальный набор

| Навык | Назначение | Priority |
|-------|------------|----------|
| git | Версионирование | critical |
| memory | Память | critical |
| shell | Команды | critical |
| web-search | Поиск | high |

### Формат навыка

```markdown
---
name: skill-name
description: "Описание"
---

# Skill Name

## Когда использовать
- Условие 1
- Условие 2

## Как использовать
1. Шаг 1
2. Шаг 2

## Примеры
...
```

## Команды

### Минимальные

| Команда | Назначение |
|---------|------------|
| `/help` | Справка |
| `/status` | Статус |
| `/reset` | Сброс контекста |

### Желательные

| Команда | Назначение |
|---------|------------|
| `/model` | Смена модели |
| `/skills` | Список навыков |
| `/memory` | Управление памятью |

## Паттерны

### Progressive Disclosure

1. **Метаданные** — всегда в контексте (~100 слов)
2. **Тело навыка** — при триггере (<5k слов)
3. **Ресурсы** — по необходимости

### Memory Management

1. **MEMORY.md** — суть, не логи
2. **Daily notes** — raw заметки
3. **Heartbeat** — периодические проверки

### Error Recovery

1. Fallback модели
2. Graceful degradation
3. User notification
